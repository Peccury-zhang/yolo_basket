# 任务：basket 掩膜底部 1/4 提取 + 上边缘线段拟合采样（v7 当前方案）

## 1. 任务目标

1. 使用 YOLO11-Seg 模型对输入图像进行 basket 实例分割。
2. 提取掩膜的**底部 1/4 区域**（红色标注）。
3. 在底部 1/4 区域的**上边缘拟合一条直线段**（黄色），并在线段上**均匀采样 10 个点**（蓝色圆点）。
4. 保存标注图像、掩膜坐标和采样点坐标。

## 2. 历史方案演进

| 版本 | 方案 | 问题 |
|------|------|------|
| v1 | 全局水平切割 `y_cut` | basket 倾斜时一侧超 1/4，另一侧不足 |
| v2 | 逐列比例裁剪 | 透视导致切割线呈梯形，不与 basket 边缘平行 |
| v3 | 旋转矩形 75% 插值 + bitwise_and | 切割区域正确 |
| v4 | 直接用 CL/CR 或最左最右上边缘点画线段 | 旋转矩形比掩膜宽导致线段超出；最左最右点在窄尖角底部导致线段偏低 |
| v5 | fitLine 拟合上边缘 + 裁掉两端 5% 尖角 | 百分位裁剪不够自适应，某些图像线段仍偏短或偏长 |
| v6 | Huber 鲁棒拟合 + 自适应离群点过滤 + 内点 x 范围 | minAreaRect 是矩形，basket 透视后为梯形，切割线与底边不平行 |
| v7 | 四边形拟合（approxPolyDP）替代旋转矩形 + 梯形侧边 75% 插值 | 上边缘更合理，但仍用拟合四边形底边裁剪，无法保证完整覆盖真实下沿 |
| v8 | 上边界直线裁切 + 下边界跟随原始掩膜 | 能覆盖真实下沿，但左右局部凸起会导致红色区域左右高度不一致 |
| **v9** | **底边鲁棒拟合 + 平行等厚梯形带 + 轻微下沿补偿** | **候选优化方案** |

## 3. 当前问题分析（v5 → v6）

### 3.1 v5 的固定百分位裁剪缺陷

v5 使用 `cv2.fitLine` 拟合上边缘后，裁掉两端 5% 的尖角点来确定线段端点。这种固定百分位裁剪在某些图像上不够自适应，导致线段仍偏短或偏长。

### 3.2 v6 修正思路

- **Huber 鲁棒拟合**：使用 `cv2.fitLine` 的 `DIST_HUBER` 距离类型，对离群点有较好的鲁棒性。
- **自适应离群点过滤**：基于法向残差的中位数 + 3×MAD 进行自适应过滤，替代固定百分位裁剪。
- **内点 x 范围**：用过滤后内点的 x 坐标范围确定线段端点，实现自适应的线段长度。

## 4. v6 当前方案

### 4.1 底部 1/4 区域提取（旋转矩形 + y 排序角点）

1. 用 `cv2.minAreaRect` 对掩膜求**最小面积旋转矩形**，`cv2.boxPoints` 获取四角点。
2. **按 y 坐标排序**：y 较小的两点为顶部，y 较大的两点为底部。
3. 在顶部两点中按 x 排序为 $TL$（左）、$TR$（右）；底部两点按 x 排序为 $BL$（左）、$BR$（右）。
4. 在 75% 处沿侧边线性插值得到切割线端点：
   - $CL = TL + 0.75 \times (BL - TL)$
   - $CR = TR + 0.75 \times (BR - TR)$
5. 由 $CL, CR, BR, BL$ 构成切割四边形，与原始掩膜 `bitwise_and` 取交集，得到底部 1/4 区域。

### 4.2 上边缘点采集

1. 扫描 `bottom_mask` **实际有像素的 x 列范围**。
2. 对每一列取最小 y 值，得到上边缘点集 `edge_points`。

### 4.3 上边缘线段拟合（Huber 鲁棒拟合 + 自适应离群点过滤）

1. 若边缘点数量 < 2，跳过该 mask。
2. 使用 `cv2.fitLine` 的 `DIST_HUBER` 距离类型拟合直线，获取方向向量 $(vx, vy)$ 和线上一点 $(x_0, y_0)$。
3. 计算方向单位向量 $\hat{d}$ 与法向量 $\hat{n} = (-\hat{d}_y, \hat{d}_x)$。
4. 计算每个边缘点到拟合线的法向距离（残差）。
5. **自适应离群点过滤**：计算残差的中位数 $r_{med}$ 和绝对偏差的中位数 MAD，阈值为 $\max(3 \times \max(MAD, 1), 5)$，过滤超过阈值的离群点。
6. 用内点的 x 坐标范围 $[x_{min}, x_{max}]$ 确定线段端点：在拟合线上找到 $x = x_{min}$ 和 $x = x_{max}$ 对应的点作为线段两端。

### 4.4 均匀采样

在拟合线段 L→R 上均匀采样 10 个点：$P_i = L + \frac{i}{9} \times (R - L)$，$i = 0, 1, ..., 9$

## 5. 效果说明

| 元素 | 颜色 | 说明 |
|------|------|------|
| 完整 basket 掩膜填充 | 绿色半透明（权重 0.3） | 模型分割输出的完整掩膜区域填充 |
| 完整 basket 掩膜轮廓 | 绿色线条（2px） | 掩膜多边形边界 |
| 底部 1/4 掩膜填充 | 红色半透明（权重 0.4） | 旋转矩形切割与掩膜的交集区域填充 |
| 底部 1/4 掩膜轮廓 | 红色线条（2px） | 对 `bottom_mask` 提取到的外轮廓逐个绘制 |
| 上边缘拟合线段 | 黄色线（2px） | Huber 鲁棒拟合，端点由内点 x 范围确定 |
| 均匀采样点 | 蓝色实心圆（半径 5px） | 每条拟合线段上等间距采样；拟合成功时绘制 10 个点 |

## 6. 输入输出

### 输入
- `source_img/{name}.{ext}`：待预测的图像（支持 png/jpg/jpeg/bmp/tif/tiff）
- `best.pt`：YOLO11-Seg 训练好的模型权重

### 输出
- `output_img/{name}_mask.{ext}`：带完整掩膜 + 底部 1/4 掩膜 + 拟合线段 + 采样点的标注图像（保留原始图像扩展名）
- `output_img/{name}_mask.txt`：掩膜坐标文件，格式如下：
  - `basket_{i}_full: x1,y1;x2,y2;...` — 完整掩膜多边形坐标
  - `basket_{i}_bottom25: x1,y1;x2,y2;...` — 底部 1/4 掩膜轮廓坐标
  - 其中 `basket_{i}_full` 的 `i` 对应模型输出的第 `i` 个掩膜实例；`basket_{i}_bottom25` 的 `i` 对应写出时遍历到的底部区域外轮廓序号，两者当前实现中不保证一一对应。
- `output_img/{name}_points.txt`：采样点坐标文件，格式如下：
  - `basket_{i}_top_edge_points:` — 标题行
  - `x,y` — 每行一个采样点坐标；拟合成功时写 10 行
  - 脚本会为每个 `bottom_mask` 都写一个标题行；若该实例无法完成直线拟合，则标题后不写坐标行

## 7. 关键技术点

- **旋转矩形 + y 排序角点**：用 `cv2.minAreaRect` 求最小旋转矩形，按 y 坐标排序区分顶部/底部角点，再按 x 区分左右。
- **75% 侧边插值切割**：沿侧边（TL→BL、TR→BR）做 75% 插值，得到切割线端点 CL、CR。
- **`cv2.bitwise_and`**：切割四边形与原始掩膜取交集，确保底部 1/4 不超出掩膜。
- **Huber 鲁棒拟合**：使用 `cv2.fitLine` 的 `DIST_HUBER` 距离类型拟合上边缘直线，对离群点有较好鲁棒性。
- **自适应离群点过滤（中位数 + 3×MAD）**：基于法向残差过滤离群点，用内点的 x 坐标范围确定线段端点。
- **等间距向量插值**：在拟合线段上均匀生成 10 个采样点。
- 支持批量处理 `source_img/` 下所有图片，多 basket 实例逐一处理。

# v7 方案：四边形拟合替代旋转矩形

## 1. v6 问题分析

### 1.1 根本原因：旋转矩形 ≠ 透视梯形

basket 经过摄像机透视投影后，在图像中呈**梯形**（上窄下宽或上宽下窄，左右两侧不平行）。然而 `cv2.minAreaRect` 拟合的是一个**矩形**（四条边严格平行/垂直），它的对边总是平行且等长。

```
  实际掩膜形状（梯形）        minAreaRect（矩形）
  
    TL───────────TR            TL'══════════TR'
   ╱               ╲           ║              ║
  ╱                 ╲          ║              ║
 ╱                   ╲         ║              ║
BL─────────────────BR          BL'══════════BR'
```

沿矩形侧边做 75% 插值时：
- 切割线 CL-CR **平行于矩形的 TL'-TR' 和 BL'-BR' 边**
- 但矩形的底边 BL'-BR' **并不平行于梯形的实际底边 BL-BR**
- 结果：切出的底部 1/4 区域的上边缘（CL-CR）与下边缘（实际掩膜底边）**不平行**，看起来不像梯形

### 1.2 具体表现

在 1_mask.png 中可以观察到：
- 红色掩膜区域的上边缘（切割线）和下边缘（掩膜底部）明显不平行
- 红色区域一侧偏厚、另一侧偏薄，不是预期的均匀梯形带状区域

## 2. v7 修正方案：四边形拟合

### 2.1 核心思路

用 `cv2.approxPolyDP` 将掩膜轮廓近似为**四边形（梯形）**，替代 `cv2.minAreaRect` 的矩形。沿梯形的实际侧边做 75% 插值，保证切割线与梯形底边平行。

### 2.2 四边形拟合算法

1. 对掩膜求最大轮廓，计算凸包 `cv2.convexHull`。
2. 使用 `cv2.approxPolyDP` 对凸包进行多边形近似，**逐步增大 epsilon**（从周长的 1% 开始），直到近似结果恰好为 **4 个顶点**。
3. 若无法精确得到 4 点（直接从 >4 跳到 <4），则取最后一次 >4 的结果，用面积最大的 4 个三角形顶点或其他策略缩减为 4 点。

### 2.3 四角点排序

1. 将 4 个顶点按 y 坐标排序，y 较小的 2 点为**顶部**，y 较大的 2 点为**底部**。
2. 顶部 2 点按 x 排序为 $TL$（左）、$TR$（右）。
3. 底部 2 点按 x 排序为 $BL$（左）、$BR$（右）。

### 2.4 底部 1/4 切割

沿梯形的实际侧边做 75% 线性插值：
- $CL = TL + 0.75 \times (BL - TL)$
- $CR = TR + 0.75 \times (BR - TR)$

由 $CL, CR, BR, BL$ 构成切割四边形，与原始掩膜 `bitwise_and` 取交集。

```
    TL───────────TR
   ╱               ╲
  ╱                 ╲
 CL─────────────────CR  ← 75% 插值（与 BL-BR 平行）
 ╱   底部 1/4 区域    ╲
BL─────────────────────BR
```

**关键保证**：由于 CL-CR 是沿梯形侧边（TL→BL、TR→BR）的 75% 插值点连线，而 TL-TR 和 BL-BR 分别是梯形的上下边，所以 CL-CR 与 BL-BR 天然近似平行（精确平行当且仅当 TL-BL 和 TR-BR 都是直线段，这在四边形拟合下成立）。

### 2.5 四边形拟合的降级策略

若 `approxPolyDP` 无法得到 4 点（掩膜形状过于不规则），则退回 v6 的 `minAreaRect` 方案作为 fallback。

### 2.6 上边缘拟合 & 采样（沿用 v6）

底部 1/4 区域提取之后的上边缘点采集、Huber 鲁棒拟合、自适应离群点过滤、均匀采样等步骤保持不变。

## 3. v7 与 v6 对比

| 对比项 | v6 | v7 |
|--------|-----|-----|
| 掩膜包围形状 | `minAreaRect` 矩形 | `approxPolyDP` 四边形（梯形） |
| 75% 插值基准 | 矩形侧边 | 梯形实际侧边 |
| CL-CR 与底边平行性 | 不保证（矩形边 ≠ 梯形边） | 保证（在梯形侧边上插值） |
| 底部 1/4 形状 | 不规则（上下不平行） | 梯形（上下近似平行） |
| 降级策略 | 无 | 退回 minAreaRect |

## 4. 需修改的代码

仅修改 `get_bottom_quarter_mask` 函数：
- 将 `cv2.minAreaRect` + `cv2.boxPoints` 替换为 `cv2.convexHull` + `cv2.approxPolyDP`
- 角点排序逻辑不变
- 75% 插值逻辑不变
- 其余函数无需修改

# v8 方案：上边界直线裁切，下边界贴合原始掩膜

## 1. v7 仍然存在的问题

v7 的底部 1/4 区域是这样得到的：

1. 先拟合一个四边形（梯形）
2. 取梯形侧边 75% 插值得到上切割线 `CL-CR`
3. 用 `CL, CR, BR, BL` 构造封闭四边形
4. 再与原始掩膜 `bitwise_and`

问题在于：这个做法不仅用到了“上切割线”，还同时用到了**拟合四边形的底边 `BL-BR` 作为下裁剪边界**。

但实际 basket 掩膜的下边缘通常不是一条理想直线，而是：

- 受实例分割轮廓影响，底边会有局部凸出/凹陷
- 左右两端会有轻微外扩
- 某些视角下前沿下边界并不完全等于拟合梯形的 `BL-BR`

因此 v7 的结果会出现：

- 红色区域的**上边界方向是对的**
- 但红色区域的**下边界被拟合梯形截断了**
- 导致红色掩膜没有完整覆盖 basket 掩膜的最下方区域

也就是说，当前真正的问题已经不是“上边界是否平行”，而是：

**下边界不应该由拟合梯形决定，而应该尽量贴合原始掩膜的真实下沿。**

## 2. v8 核心思路

将“底部 1/4 区域”改为：

- **上边界**：仍然使用四边形侧边 75% 插值得到的直线 `CL-CR`
- **下边界**：不再使用拟合梯形底边 `BL-BR` 裁剪，而是**直接保留原始掩膜在该直线下方的全部区域**

这相当于把当前方案从“梯形裁剪”改成“**半平面裁剪**”：

- 不再构造 `CL, CR, BR, BL` 的封闭梯形
- 只保留 `CL-CR` 下方的掩膜部分

这样得到的红色区域会：

- 上边缘保持一条稳定直线
- 下边缘自动跟随原始 basket 掩膜
- 尽量贴合甚至完全覆盖 basket 的下方区域

## 3. v8 几何定义

### 3.1 上切割线保持不变

仍然先通过四边形拟合得到 $TL, TR, BL, BR$，然后：

- $CL = TL + 0.75 \times (BL - TL)$
- $CR = TR + 0.75 \times (BR - TR)$

上切割线定义为经过 `CL` 和 `CR` 的直线。

### 3.2 保留下半平面中的掩膜像素

设直线方向从 `CL -> CR`，对图像中的任意像素点 $P=(x,y)$，判断它是否位于该直线的“下方”。

可用二维叉积符号判断：

$$
s(P) = (CR_x-CL_x)(y-CL_y) - (CR_y-CL_y)(x-CL_x)
$$

对图像坐标系（y 向下）而言，只需统一选定一侧为“保留侧”，即可得到一张 `below_line_mask`。

最终区域：

$$
bottom\_mask = full\_mask \cap below\_line\_mask
$$

这里的关键变化是：

- v7: `full_mask ∩ trapezoid_mask`
- v8: `full_mask ∩ below_line_mask`

v8 不再限制下边界必须落在 `BL-BR` 之上，因此真实下沿都会被保留下来。

## 4. 与 v7 的本质差异

### 4.1 v7 是“封闭四边形裁剪”

```
    TL──────────TR
   ╱            ╲
  ╱              ╲
 CL──────────────CR
 ╱                ╲
BL────────────────BR
```

红色区域被限制在这个封闭梯形内部，超出 `BL-BR` 的真实掩膜下沿会被截掉。

### 4.2 v8 是“单条上边界切一刀”

```
完整掩膜：

      ┌────────────┐
     ╱              ╲
    ╱                ╲
   ╱                  ╲
  ╱____________________╲

上切割线：
  ───────────────

保留结果：只保留该线以下的原始掩膜
```

因此：

- 上边界可控
- 下边界完全贴合原始掩膜
- 红色区域更符合“basket 下方区域”的语义

## 5. 预期效果

v8 实现后，1_mask.png 这类图像中的红色掩膜会表现为：

- 顶边仍然是一条近似平行于 basket 前沿的直线
- 底边不再是人为拟合的直线，而是贴合绿色 basket 掩膜的真实最下缘
- 红色区域会更“厚”一些，能够覆盖当前遗漏的下沿区域

从任务目标上看，v8 更符合“**尽量贴合或者覆盖 basket 掩膜下方**”这一要求。

## 6. 实现方式

代码上只需要修改 `get_bottom_quarter_mask` 的裁剪逻辑：

1. 保留当前的四边形拟合逻辑，用于求 `TL, TR, BL, BR`
2. 保留当前的 75% 插值逻辑，用于求 `CL, CR`
3. 删除 `cut_quad = [CL, CR, BR, BL]` 的封闭梯形裁剪方式
4. 改为生成一张“直线下方区域”二值 mask
5. 用 `full_mask` 与该 mask 做 `bitwise_and`

即：

- 保留“如何确定上边界”的部分
- 替换“如何得到下方区域”的部分

## 7. 推荐实现版本

推荐优先采用以下版本：

- 四边形拟合确定方向
- `CL-CR` 作为唯一切割边界
- 红色区域定义为“原始掩膜中位于 `CL-CR` 下方的全部像素”

这是当前最符合需求的定义，因为它同时满足：

- 上边界稳定
- 下边界贴合真实 mask
- 不会再漏掉 basket 最下方的有效区域

# v9 方案：底边鲁棒拟合 + 平行等厚梯形带

## 1. v8 仍然存在的问题

v8 解决了“红色区域没有覆盖到 basket 下沿”的问题，但它引入了另一个视觉问题：

- 红色区域的**上边界**是一条直线
- 红色区域的**下边界**直接跟随原始 mask

而实例分割得到的原始下边界通常并不平整，尤其在左右两端会出现：

- 局部凸起
- 侧面边缘混入
- 一端略高、一端略低

因此在 1_mask.png、18_mask.png 这类图像中，会看到：

- 红色区域整体虽然覆盖了下沿
- 但左右两侧的“高度”不一致
- 视觉上不像一个规整的梯形带状区域

换句话说，v8 的问题已经从“覆盖不够”变成了“**覆盖是够了，但形状不够规整**”。

## 2. v9 核心思路

v9 的目标是同时满足两个条件：

1. 红色区域尽量覆盖 basket 下沿
2. 红色区域左右两侧高度尽量一致，整体更像规则梯形

为此，v9 不再直接使用原始 mask 的下边界作为最终下边界，而是采用：

- **底边鲁棒拟合**：从原始 mask 的下沿点中拟合出一条稳定的“前沿底边”
- **平行等厚带**：上边界与下边界保持平行，厚度按法向距离定义为常数
- **轻微下沿补偿**：为避免再次漏掉真实下沿，可在拟合底边下方加一个很小的补偿带

这相当于从 v8 的“上直下弯”区域，改成“**上下都近似直线、并且互相平行**”的规则梯形区域。

## 3. v9 几何定义

### 3.1 拟合 basket 的前沿底边

对完整掩膜 `full_mask`：

1. 在每个 x 列上取最底部像素，形成下包络点集 `bottom_edge_points`
2. 只保留中间主体区域的点，剔除左右两端容易混入侧边的位置
3. 使用 Huber 或 RANSAC 对这些点做鲁棒直线拟合，得到 basket 的**前沿底边** `bottom_line`

这里的核心是：

- 不直接信任 raw mask 的锯齿状下沿
- 而是提取它的“主趋势线”

### 3.2 用法向距离定义等厚梯形带

设拟合得到的前沿底边为 `bottom_line`，其单位法向量为 $\hat{n}$。

令目标厚度为 $d$，则顶部切割线不再直接使用 v8 的 `CL-CR`，而是改为：

- 与 `bottom_line` **平行**
- 沿法向量 $-\hat{n}$ 向上偏移距离 $d$

即得到新的 `top_line`。

这样定义后：

- 上边界和下边界天然平行
- 梯形带在左右两侧的高度一致
- 不再出现一边厚一边薄的问题

### 3.3 厚度 d 的确定方式

厚度 $d$ 不建议简单用图像坐标的 y 差，而应使用**到底边的法向距离**。

推荐两种实现方式：

1. 用四边形侧边 75% 插值得到的 `CL/CR`，计算它们到底边拟合线的法向距离，取平均值作为 $d$
2. 用四边形上下边之间的法向高度，乘以 0.25，作为底部 1/4 的目标厚度

相比 v8，关键改动是：

- v8：上边界由侧边插值得到，下边界跟 raw mask
- v9：先拟合下边界，再生成与之平行的上边界

## 4. 侧边约束

为保持区域仍然是梯形，而不是无限平行带，v9 仍然保留左右侧边约束：

- 左边界使用拟合四边形的左侧边
- 右边界使用拟合四边形的右侧边

最终区域由以下四条边围成：

1. 左侧边：梯形左边
2. 右侧边：梯形右边
3. 下边界：鲁棒拟合得到的 `bottom_line`
4. 上边界：与 `bottom_line` 平行、向上偏移 $d$ 的 `top_line`

这样得到的是一个真正意义上的**规则梯形带**。

## 5. 下沿补偿策略

如果只使用拟合的 `bottom_line` 作为下边界，可能又会重新出现“少量下沿没有覆盖”的问题。

因此 v9 增加一个小补偿：

- 在 `bottom_line` 下方沿法向再放宽一个很小的距离 $\delta$，例如 3 到 6 像素
- 或者把最终梯形带与“底边附近的原始 mask 薄层”做并集

这样可以兼顾两点：

- 主体区域保持规整，左右高度一致
- 又不会因为底边拟合过于理想化而漏掉真实 basket 下沿

## 6. 与 v8 的本质差异

### 6.1 v8：覆盖优先

- 上边界是直线
- 下边界跟随原始 mask
- 优点：覆盖完整
- 缺点：左右高度不一致，局部形状不规整

### 6.2 v9：规整优先，同时保留少量补偿

- 下边界先鲁棒拟合成直线
- 上边界与下边界平行
- 厚度用法向距离定义为常数
- 最后只给下边界增加一层很薄的补偿

因此 v9 兼顾：

- 梯形规整度
- 左右高度一致性
- 对真实下沿的覆盖能力

## 7. 预期效果

在 1_mask.png 和 18_mask.png 这类图像上，v9 预期会比 v8 更稳定：

- 红色区域左右厚度更一致
- 顶边、底边更接近平行
- 两端不会因为原始 mask 的局部凸起而一高一低
- 同时还能保留对 basket 下沿的基本覆盖

## 8. 代码修改建议

若实现 v9，主要修改 `get_bottom_quarter_mask`：

1. 新增 `_get_bottom_edge_points(full_mask)`：逐列提取最底部像素
2. 新增 `_fit_bottom_edge_line(points)`：对下沿点做鲁棒拟合
3. 新增 `_build_parallel_band_mask(...)`：构造平行等厚梯形带
4. 保留当前四边形拟合逻辑，用于提供左右侧边约束
5. 删除“下边界直接跟随原始 mask”的主逻辑，改为“拟合底边 + 小补偿”

## 9. 推荐实现版本

推荐的 v9 落地版本为：

- 四边形拟合负责确定左右方向
- 下沿点鲁棒拟合负责确定前沿底边
- 上边界由底边平行偏移得到
- 最终区域为规则梯形带
- 在底边处增加 3 到 6 像素的补偿余量

这个版本比 v8 更适合当前诉求，因为它解决的是：

- v8 的覆盖问题已经解决
- 当前更主要的是提升红色区域的**几何规整性**
