# 任务：basket 掩膜底部 1/4 提取 + 上边缘线段拟合采样（v7 最终方案）

## 1. 任务目标

1. 使用 YOLO11-Seg 模型对输入图像进行 basket 实例分割。
2. 提取掩膜的**底部 1/4 区域**（红色标注）。
3. 在底部 1/4 区域的**上边缘拟合一条直线段**（黄色），并在线段上**均匀采样 10 个点**（蓝色圆点）。
4. 保存标注图像、掩膜坐标和采样点坐标。

## 2. 历史方案演进

| 版本 | 方案 | 问题 |
|------|------|------|
| v1 | 全局水平切割 `y_cut` | basket 倾斜时一侧超 1/4，另一侧不足 |
| v2 | 逐列比例裁剪 | 透视导致切割线呈梯形，不与 basket 边缘平行 |
| v3 | 旋转矩形 75% 插值 + bitwise_and | 切割区域正确 |
| v4 | 直接用 CL/CR 或最左最右上边缘点画线段 | 旋转矩形比掩膜宽导致线段超出；最左最右点在窄尖角底部导致线段偏低 |
| v5 | fitLine 拟合上边缘 + 裁掉两端 5% 尖角 | 百分位裁剪不够自适应，某些图像线段仍偏短或偏长 |
| v6 | Huber 鲁棒拟合 + 自适应离群点过滤 + 内点 x 范围 | 按 y 排序角点在旋转矩形倾斜时混淆不同边的角点，底部 1/4 呈梯形而非平行四边形；拟合线方向自由，不平行于底边 |
| v7 | 基于边识别底边 + 平行四边形切割 + 固定底边方向拟合 | **当前方案** |
| ~~v8~~ | ~~凸包四边形近似 + 沿实际侧边插值~~ | 已撤销：approxPolyDP 近似不稳定 |

## 3. 当前问题分析（v6 → v7）

### 3.1 v6 的角点排序缺陷

v6 对旋转矩形的 4 个角点按 **y 坐标排序**，取 y 较小的两点为"顶部"、y 较大的两点为"底部"，再按 x 区分左右。

当旋转矩形倾斜较大时，这种排序会将**不同边**的角点混为同一组，导致 CL-CR 不平行于 BL-BR → 切割四边形为梯形。

### 3.2 v7 修正思路

- **基于边识别**：利用 `cv2.boxPoints` 返回的角点顺序（相邻角点共享一条边），找到 y 均值最大的**底边**，再通过邻接关系正确配对 TL↔BL、TR↔BR。
- **平行四边形切割**：沿正确的侧边插值，CL-CR 必然平行于 BL-BR。
- **固定方向拟合**：拟合线方向锁定为底边方向 $\vec{d} = BR - BL$，仅拟合垂直偏移量，确保线段平行于底边。

## 4. v7 最终方案

### 4.1 底部 1/4 区域提取（基于边识别的平行四边形切割）

1. 用 `cv2.minAreaRect` 对掩膜求**最小面积旋转矩形**，`cv2.boxPoints` 获取四角点。
2. **基于边识别底边**：`cv2.boxPoints` 返回的 4 个角点按顺序相邻（即 `box[0]-box[1]`、`box[1]-box[2]`、`box[2]-box[3]`、`box[3]-box[0]` 各为一条边）。计算每条边两端点的**平均 y**，取平均 y 最大的边为**底边**。
3. 底边两端点按 x 排序为 $BL$（左）、$BR$（右）。通过旋转矩形的邻接关系找到另外两个角点，分别为与 $BL$ 相邻的 $TL$、与 $BR$ 相邻的 $TR$。
4. 在 75% 处沿**正确的侧边**线性插值得到切割线端点：
   - $CL = TL + 0.75 \times (BL - TL)$
   - $CR = TR + 0.75 \times (BR - TR)$
5. 由 $CL, CR, BR, BL$ 构成**平行四边形**（$CL$-$CR$ 必然平行于 $BL$-$BR$），与原始掩膜 `bitwise_and` 取交集，得到底部 1/4 区域。
6. 同时保留底边方向向量 $\vec{d} = BR - BL$，供后续拟合使用。

### 4.2 上边缘点采集

1. 扫描 `bottom_mask` **实际有像素的 x 列范围**。
2. 对每一列取最小 y 值，得到上边缘点集 `edge_points`。

### 4.3 上边缘线段拟合（固定底边方向 + 拟合垂直偏移）

1. 若边缘点数量 < 2，跳过该 mask。
2. 直线方向**锁定为底边方向** $\vec{d} = BR - BL$，归一化为单位向量 $\hat{d}$。
3. 计算底边法向量 $\hat{n} = (-\hat{d}_y, \hat{d}_x)$。若 $\hat{n}$ 的 y 分量 > 0（指向下方），取反以确保指向上方。
4. 将每个上边缘点投影到法向量方向，得到法向距离 $t_i = (P_i - BL) \cdot \hat{n}$。
5. 取法向距离的**中位数** $t_{med}$ 作为拟合线的偏移量（中位数对离群点鲁棒）。
6. 确定线段端点：投影到底边方向 $s_i = (P_i - BL) \cdot \hat{d}$，用**自适应离群点过滤**（中位数 + 3×MAD，基于法向残差 $|t_i - t_{med}|$）过滤侧面边缘离群点，取内点的 $s$ 范围 $[s_{min}, s_{max}]$：
   - $L = BL + s_{min} \cdot \hat{d} + t_{med} \cdot \hat{n}$
   - $R = BL + s_{max} \cdot \hat{d} + t_{med} \cdot \hat{n}$

### 4.4 均匀采样

在拟合线段 L→R 上均匀采样 10 个点：$P_i = L + \frac{i}{9} \times (R - L)$，$i = 0, 1, ..., 9$

## 6. 输入输出

### 输入
- `source_img/{name}.{ext}`：待预测的图像（支持 png/jpg/jpeg/bmp/tif/tiff）
- `best.pt`：YOLO11-Seg 训练好的模型权重

### 输出
- `output_img/{name}_mask.{ext}`：带完整掩膜 + 底部 1/4 掩膜 + 拟合线段 + 采样点的标注图像（保留原始图像扩展名）
- `output_img/{name}_mask.txt`：掩膜坐标文件，格式如下：
  - `basket_{i}_full: x1,y1;x2,y2;...` — 完整掩膜多边形坐标
  - `basket_{i}_bottom25: x1,y1;x2,y2;...` — 底部 1/4 掩膜轮廓坐标
  - 其中 `basket_{i}_full` 的 `i` 对应模型输出的第 `i` 个掩膜实例；`basket_{i}_bottom25` 的 `i` 对应写出时遍历到的底部区域外轮廓序号，两者当前实现中不保证一一对应。
- `output_img/{name}_points.txt`：采样点坐标文件，格式如下：
  - `basket_{i}_top_edge_points:` — 标题行
  - `x,y` — 每行一个采样点坐标；拟合成功时写 10 行
  - 脚本会为每个 `bottom_mask` 都写一个标题行；若该实例无法完成直线拟合，则标题后不写坐标行

## 7. 关键技术点

- **基于边识别底边**：利用 `cv2.boxPoints` 角点的顺序邻接关系，找到平均 y 最大的边作为底边，避免 y 排序在倾斜矩形下混淆角点。
- **平行四边形切割**：沿正确配对的侧边（TL→BL、TR→BR）做 75% 插值，CL-CR 必然平行于 BL-BR，切割区域为平行四边形。
- **`cv2.bitwise_and`**：切割四边形与原始掩膜取交集，确保底部 1/4 不超出掩膜。
- **固定底边方向拟合**：拟合线方向锁定为 $\vec{d} = BR - BL$，仅拟合法向偏移量（中位数），确保线段平行于底边。
- **自适应离群点过滤（中位数 + 3×MAD）**：在法向残差上过滤侧面边缘离群点，用内点在底边方向的投影范围确定线段端点。
- **等间距向量插值**：在拟合线段上均匀生成 10 个采样点。
- 支持批量处理 `source_img/` 下所有图片，多 basket 实例逐一处理。
