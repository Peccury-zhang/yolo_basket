# 任务：basket 掩膜底部 1/4 提取 + 上边缘线段拟合采样（v6 最终方案）

## 1. 任务目标

1. 使用 YOLO11-Seg 模型对输入图像进行 basket 实例分割。
2. 提取掩膜的**底部 1/4 区域**（红色标注）。
3. 在底部 1/4 区域的**上边缘拟合一条直线段**（黄色），并在线段上**均匀采样 10 个点**（蓝色圆点）。
4. 保存标注图像、掩膜坐标和采样点坐标。

## 2. 历史方案演进

| 版本 | 方案 | 问题 |
|------|------|------|
| v1 | 全局水平切割 `y_cut` | basket 倾斜时一侧超 1/4，另一侧不足 |
| v2 | 逐列比例裁剪 | 透视导致切割线呈梯形，不与 basket 边缘平行 |
| v3 | 旋转矩形 75% 插值 + bitwise_and | 切割区域正确 |
| v4 | 直接用 CL/CR 或最左最右上边缘点画线段 | 旋转矩形比掩膜宽导致线段超出；最左最右点在窄尖角底部导致线段偏低 |
| v5 | fitLine 拟合上边缘 + 裁掉两端 5% 尖角 | 百分位裁剪不够自适应，某些图像线段仍偏短或偏长 |
| **v6** | **Huber 鲁棒拟合 + 自适应离群点过滤 + 内点 x 范围** | **当前方案** |

## 3. 当前问题分析

底部 1/4 区域（`bottom_mask`）= 旋转矩形切割四边形 ∩ 原始掩膜。由于两者形状不同，交集区域的左右两端会形成**窄尖角**。逐列扫描上边缘时，这些尖角处的边缘点实际是**侧面边缘**（y 值快速增大，向下弯曲），而非真正的顶部水平边缘。

```
          真正的顶部边缘（近似水平）
          ┌─────────────────────┐
    ╱    ╱                       ╲    ╲
   ╱ 侧 ╱                         ╲ 侧 ╲
  ╱ 面  ╱     红色掩膜区域           ╲ 面  ╲
 ╱ 边  ╱                             ╲ 边  ╲
╱ 缘  ╱───────────────────────────────╲ 缘  ╲
      BL                               BR
```

直接用所有上边缘点的 x 范围 → 线段过长，超出红色掩膜顶边；
用 CL/CR 的 x 范围 → 可能偏短（CL/CR 是旋转矩形角点，不一定对应掩膜边界）。

## 4. v6 最终方案

### 4.1 底部 1/4 区域提取（沿用 v3）

1. 用 `cv2.minAreaRect` 对掩膜求**最小面积旋转矩形**，`cv2.boxPoints` 获取四角点。
2. 四角点按 y 排序分为顶部（TL, TR）和底部（BL, BR），再按 x 区分左右。
3. 在 75% 处线性插值得到切割线端点：
   - $CL = TL + 0.75 \times (BL - TL)$
   - $CR = TR + 0.75 \times (BR - TR)$
4. 由 CL, CR, BR, BL 构成四边形，与原始掩膜 `bitwise_and` 取交集，得到底部 1/4 区域。
5. 脚本内部会同时保留 `CL`、`CR`，与 `bottom_mask` 一起进入后续流程；当前上边缘扫描阶段实际使用的是 `bottom_mask` 的像素范围，不依赖 `CL`、`CR` 的 x 坐标。

### 4.2 上边缘点采集

1. 扫描 `bottom_mask` **实际有像素的 x 列范围**（不依赖 CL/CR 坐标）。
2. 对每一列取最小 y 值，得到上边缘点集 `edge_points`。

### 4.3 上边缘线段拟合（v6 核心改进：两阶段鲁棒拟合）

**第一阶段：鲁棒拟合 + 离群点过滤**

1. 若边缘点数量 < 2，跳过该 mask。
2. 用 `cv2.fitLine` 的 **Huber 距离**（`cv2.DIST_HUBER`）对全部上边缘点做鲁棒拟合，得到初始直线方向和位置。若拟合方向向量 vx ≈ 0（近似竖直），则强制 slope = 0。Huber 距离对离群点不敏感，避免侧面边缘点拉偏方向。
3. 计算每个边缘点到拟合线的**垂直残差**：$r_i = |y_i - (cy + slope \times (x_i - cx))|$
4. 用**自适应阈值**过滤离群点：
   - $threshold = \max(median(r) + 3 \times \max(MAD, 1), 5)$
   - 其中 $MAD = median(|r_i - median(r)|)$
   - MAD 下限为 1（防止所有点共线时 MAD=0 导致阈值过小）
   - threshold 下限为 5 像素
   - 残差 ≤ threshold 的点为**内点**（顶部水平边缘），其余为**离群点**（侧面边缘）

**第二阶段：精确拟合 + 端点确定**

4. 若内点数量 < 2，则回退使用全部边缘点。
5. 用**内点**重新做 L2 最小二乘拟合（`cv2.DIST_L2`），得到精确的直线参数。若拟合方向向量 vx ≈ 0（近似竖直），则强制 slope = 0。
6. 用**内点的 x 最小值和最大值**计算直线端点 → 线段恰好覆盖顶部水平边缘，不受侧面尖角影响。

```
  全部边缘点：  ·  ·  ·  ● ● ● ● ● ● ● ● ● ●  ·  ·
                ↑ 离群点（侧面）   ↑ 内点（顶边）    ↑ 离群点（侧面）
  拟合线段：              L●━━━━━━━━━━━━━━●R
```

### 4.4 均匀采样

在拟合线段 L→R 上均匀采样 10 个点：$P_i = L + \frac{i}{9} \times (R - L)$，$i = 0, 1, ..., 9$

## 5. 效果说明

| 元素 | 颜色 | 说明 |
|------|------|------|
| 完整 basket 掩膜填充 | 绿色半透明（权重 0.3） | 模型分割输出的完整掩膜区域填充 |
| 完整 basket 掩膜轮廓 | 绿色线条（2px） | 掩膜多边形边界 |
| 底部 1/4 掩膜填充 | 红色半透明（权重 0.4） | 旋转矩形切割与掩膜的交集区域填充 |
| 底部 1/4 掩膜轮廓 | 红色线条（2px） | 对 `bottom_mask` 提取到的外轮廓逐个绘制 |
| 上边缘拟合线段 | 黄色线（2px） | 两阶段鲁棒拟合，端点由内点 x 范围确定 |
| 均匀采样点 | 蓝色实心圆（半径 5px） | 每条拟合线段上等间距采样；拟合成功时绘制 10 个点 |

## 6. 输入输出

### 输入
- `source_img/{name}.{ext}`：待预测的图像（支持 png/jpg/jpeg/bmp/tif/tiff）
- `best.pt`：YOLO11-Seg 训练好的模型权重

### 输出
- `output_img/{name}_mask.{ext}`：带完整掩膜 + 底部 1/4 掩膜 + 拟合线段 + 采样点的标注图像（保留原始图像扩展名）
- `output_img/{name}_mask.txt`：掩膜坐标文件，格式如下：
  - `basket_{i}_full: x1,y1;x2,y2;...` — 完整掩膜多边形坐标
  - `basket_{i}_bottom25: x1,y1;x2,y2;...` — 底部 1/4 掩膜轮廓坐标
  - 其中 `basket_{i}_full` 的 `i` 对应模型输出的第 `i` 个掩膜实例；`basket_{i}_bottom25` 的 `i` 对应写出时遍历到的底部区域外轮廓序号，两者当前实现中不保证一一对应。
- `output_img/{name}_points.txt`：采样点坐标文件，格式如下：
  - `basket_{i}_top_edge_points:` — 标题行
  - `x,y` — 每行一个采样点坐标；拟合成功时写 10 行
  - 脚本会为每个 `bottom_mask` 都写一个标题行；若该实例无法完成直线拟合，则标题后不写坐标行

## 7. 关键技术点

- **`cv2.minAreaRect` + `cv2.boxPoints`**：获取旋转矩形角点，提供 basket 倾斜方向。
- **75% 线性插值**：沿旋转矩形短边在 3/4 处生成与 basket 平行的切割线。
- **`cv2.bitwise_and`**：切割四边形与原始掩膜取交集，确保底部 1/4 不超出掩膜。
- **Huber 鲁棒拟合**：第一阶段用 `cv2.DIST_HUBER`，对侧面边缘离群点不敏感。
- **自适应离群点过滤（中位数 + 3×MAD）**：自动区分顶部边缘（内点）和侧面边缘（离群点），无需硬编码百分位。
- **内点 x 范围端点**：拟合线段恰好覆盖红色掩膜顶部水平边缘，不短不长。
- **等间距向量插值**：在拟合线段上均匀生成 10 个采样点。
- **近竖直拟合保护**：当 `vx` 过小，脚本将斜率强制为 0，避免数值不稳定。
- 支持批量处理 `source_img/` 下所有图片，多 basket 实例逐一处理。
